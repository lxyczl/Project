# 中文学术论文降重

## 核心流程

### 反馈闭环（每次改写必须遵循）

```
获取建议 → 改写 → 分析+记录 → 记录技巧 → 自动学习
```

1. **获取建议**：改写前运行 `$PY scripts/rewrite_with_feedback.py suggest <学科> <强度>`，返回 JSON 包含：
   - `preferred_vocabulary`：历史高分替换词对（如"研究->探究"），改写时优先使用
   - `effective_techniques`：成功率 ≥70% 的技巧，改写时优先选择
   - `intensity_multiplier`：强度乘数（>1 表示需要加强改写力度，<1 表示可以减弱）
   - `domain_issues`：该学科的常见问题，改写时注意避免
   - `new_terms_to_preserve`：用户报告过的遗漏术语，改写时标记为不可改
2. **改写**：结合建议和下方规则执行改写
3. **分析+记录**：改写后运行 `$PY scripts/rewrite_with_feedback.py analyze <原文文件> <改写文件> <学科> <强度>`，自动计算相似度、记录会话、输出自动评估和学习建议
4. **记录技巧**：将本次使用的技巧记录到会话：`$ scripts/rewrite_with_feedback.py changes <会话ID> '[{"type":"技巧名","original":"原文片段","rewritten":"改写片段"}]'`
5. **自动学习**：运行 `$PY scripts/rewrite_with_feedback.py learn <会话ID>`，系统基于客观指标（连续匹配字数、三元组重叠率）自动判定成功/失败并更新策略

**自动评估标准**（基于知网查重规则，无需用户打分）：
- `excellent`：连续匹配 < 7 字 且 三元组 < 10% → 记为成功，略微降低强度
- `success`：连续匹配 < 10 字 且 三元组 < 20% → 记为成功
- `warning`：连续匹配 10-12 字 或 三元组 ≥ 20% → 记为失败，加强改写力度
- `fail`：连续匹配 ≥ 13 字 → 直接不通过，必须继续改写

**句子级热点定位**：`analyze` 命令现在返回 `hot_sentences`（需要重点改写的句子）和 `needs_iteration` 标记。当 `needs_iteration` 为 true 时，应自动对热点句子进行针对性改写（最多 3 轮）。

**手动反馈（可选）**：用户如果想提供更详细的主观评价，运行 `$PY scripts/rewrite_with_feedback.py feedback <会话ID> <词汇分> <结构分> <术语分> <总分>`（各 1-5 分）。大多数情况下自动学习已足够。

**changes 命令的技巧类型**（type 字段必须与以下值严格一致）：
通用技巧：`句式重组`、`主被动转换`、`拆分长句`、`合并短句`、`引用位置移动`、`同义词替换`、`增加修饰语`、`删除冗余`、`调整语序`、`添加过渡词`、`具体化`、`抽象化`、`因果倒置`、`条件重组`、`否定反转`
中文特色：`四字词语重组`、`文言成分替换`、`句末语气词调整`、`把字句/被字句转换`、`话题-评论结构重组`、`量词替换`、`并列结构重组`、`否定表达重组`、`数量表达重组`、`时间表达重组`

### 场景1：用户发文本
1. 获取建议 → 2. 按规则改写 → 3. 分析结果（含句子级热点和 needs_iteration 标记）→ 4. 如果 needs_iteration 为 true：定位热点句子，使用 suggested_techniques 针对性改写，再次分析验证（最多 3 轮；3 轮后仍 fail 则返回当前结果并提示需手动调整）→ 5. 返回最终结果 → 6. 询问满意度 → 7. 记录反馈

### 场景2：用户给文件
1. 获取建议 → 2. 分段改写（每段 ≤500 字）→ 3. 分析每段结果（含句子级热点）→ 4. 对 needs_iteration 的段落自动迭代改写（最多 3 轮）→ 5. 合并结果+报告 → 6. 询问满意度 → 7. 记录反馈

### 场景3：用户反馈/迭代
1. 根据反馈调整强度或技巧 → 2. 重新改写 → 3. 分析 → 4. 再次收集反馈

### 查看学习报告
运行 `$PY scripts/rewrite_with_feedback.py report` 查看当前学习到的策略（词汇偏好、技巧有效性、领域模式等）

## 必须保留（不能改）

专业术语、引用 [1]、公式、数字、单位、地名、模型名、人名、缩写

## 关键规则

1. **不能连续13个字和原文相同** — 最关键（知网查重规则）
2. **每句话至少用2种技巧**
3. **保持学术正式语体**
4. **不改变原意**

---

## 改写技巧概览

两类技巧互补使用。详细说明和示例见 `references/techniques.md`（通用技巧）和 `references/chinese-specific.md`（中文特色技巧）。

### 通用技巧（15种，见 techniques.md）

**第一梯队：优先使用** — 句式重组、主被动转换、拆分长句、合并短句、引用位置移动

**第二梯队：常用** — 同义词替换、增加修饰语、删除冗余、调整语序、添加过渡词

**第三梯队：辅助** — 具体化、抽象化、因果倒置、条件重组、否定反转

### 中文特色技巧（10种，见 chinese-specific.md）

四字词语重组、文言成分替换、句末语气词调整、把字句/被字句转换、话题-评论结构重组、量词替换、并列结构重组、否定表达重组、数量表达重组、时间表达重组

## 复杂句处理

**长句拆分**：超过40字，考虑拆成2句

**嵌套从句重组**：多层嵌套简化为并列结构

**并列结构变换**：重复主语合并为并列结构

## 句型模板

| 场景 | 改写模式 |
|------|---------|
| 研究表明 | 相关研究证实/已有研究揭示/众多学者指出 |
| 具有重要意义 | 在...中发挥关键作用/对...具有重要影响/是...的核心因素 |
| 因为...所以 | 由于...，因此/鉴于...，故而/基于...，从而 |

## 反面教材

- ❌ 只换词不改结构（相似度还是高）
- ❌ 改变原意（加了"可能"等限定词）
- ❌ 破坏专业术语（把"生态系统服务"改成"生态服务"）
- ❌ 破坏引用格式（[1] 变成 [ 1 ]）
- ❌ 过度拆分（句子太碎不流畅）
- ❌ 使用口语化表达（"很多"→"老多"）
- ❌ 欧化表达（直译导致"的的的"等不自然结构）
- ❌ 滥用被字句（中文被字句有时别扭）
- ❌ 滥用文言成分（过度使用"之乎者也"）
- ❌ 删除必要成分（把"的"删掉导致语法错误）

## 示例

**示例1**：句式重组+同义词替换
> 原文：研究表明，城市化进程的加快导致了生态环境的恶化。
> 改写：生态环境的恶化与城市化进程的加速密切相关，这一关系已得到众多研究的证实。

**示例2**：句式重组+增加修饰语
> 原文：研究者发现地下水位下降是导致植被退化的主要原因。
> 改写：大量研究表明，植被退化与地下水位下降之间存在密切关联。

**示例3**：拆分长句+引用移动
> 原文：由于气候变化和人类活动的双重影响，该地区的生态系统服务功能显著下降[1]。
> 改写：该地区的生态系统服务功能出现显著下降。根据[1]的研究，这一现象与气候变化和人类活动的双重影响密切相关。

## 边界情况

详细处理方式见 `references/edge_cases.md`。以下为速查：

| 情况 | 处理方式 |
|------|---------|
| 短文本（<50字） | 告知用户太短，建议提供更长段落 |
| 非学术文本 | 告知技能不适用，如用户坚持仍尝试 |
| 已低相似度 | 先计算，如果已经很低则告知不需要改写 |
| 纯英文文本 | 告知用户本技能仅支持中文 |
| 大量公式/引用 | 标记不改，只改其他部分 |
| 超长文本（>1000字） | 建议分段，每段不超过500字 |
| 列表/枚举 | 保留结构，只改内容 |
| 直接引语 | 引号内不改，只改转述部分 |
| 术语密集 | 标记术语不改，只改连接词和动词 |
